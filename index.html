Chapter 1  Obtaining Data

input <- readFile "input.txt"

Data is everywhere, logging is cheap, and analysis is inevitable. One of the most fundamental concepts of this chapter is about gathering useful data. After building a collection, we must learn to represent this data in code. The primary focus will be on first obtaining data, and in later chapters we will enumerate ways to represent it. This chapter will cover the following essential methods for working with Haskell input.

  • Gathering various sources to build a corpus
  • Reading raw text from a file path
  • Reading and representing a CSV file
  • Parsing a JSON file by using aeson
  • Parsing XML using the HXT package
  • Reading table rows from HTML
  • Downloading data from HTTP GET requests
  • Downloading data from HTTP POST requests
  • Mining data from online directory searches
  • Querying MongoDB for data (local)
  • Querying MongoDB for data (remote)

Chapter 7  Parallelism and Concurrency

One of the greatest accomplishments in the study of data analysis is the intelligent approach to parallel and concurrent design. As we collect more and more data, we are able to discover more and more patterns. However, this comes at a price of time and space. More data may take more time to compute or more space in terms of memory. This is a very real problem which this chapter will try to solve.

  • Parallelizing pure functions using the Par monad
  • Mapping over a list in parallel
  • Accessing tuple elements in parallel
  • Implementing MapReduce to count word frequencies
